# Knowledge network conventions

Last updated: 2026-07-20

This directory is a curated operating network, not a transcript archive. Read `../AGENTS.md` first, use its task router, then open only relevant documents.

Status values are `green` (working), `yellow` (working with pending work), `red` (broken), `unknown` (not recently verified), and `planned` (not provisioned). Stable facts live under `components/`; live state lives under `status/`; hard-won traps live under `gotchas/`; repeatable procedures live under `workflows/`.

Update component briefs only when architecture or conventions change. Update status documents every 2–3 turns and at session end. Add decisions only when the reason will matter to future work. Never copy secrets, tokens, account identifiers, or private deployment metadata into this network.
